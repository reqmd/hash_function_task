# Создание хэш-функции 

## В файле hash_function_task.py приведен пример работы хэш таблицы с простейшей хэш функцией деления. 
### В классе реализованны такие методы как:

+ _hash_function(self, key): Сама хэш функция которая преобразует числовое значение в ключ для значения в таблице
+ insert(self, key, value): Добавляет элемент в хэш функцию
+ search(self, key): Ищет значение по ключу
+ delete(self, key): Удаляет значение по ключу
+ fill_random(self, num_items, collision_keys): Заполняет хэш таблицу случайными числами
+ check_bucket_uniformity(self): Считает метрики заполнености таблицы и бакетов.

### Задача заключается в том, чтобы реализовать другую хэш функцию, например SHA1 или MD5.
#### Зачем это нужно?
Если вы запустите файл *main.py*, он создаст вам хэш таблицу, на основе большого кол-ва случайных значений. Из-за того, что наша хэш функция достаточно простая, она не сможет правильно распределить случайные данные. В этом мы можем убедиться если посмотрим содержимое *HashTable.table* после заполнения таблицы случайными данными. Мы увидим что много бакетов остаются пустыми, а некоторые содержат в себе очень большое количество, так называемых, коллизий. Также, для того чтобы не выводить постоянно содержимое таблицы, сделан метод *check_bucket_uniformity*, которые показывает нужные нам статистики.\

При создании своей хэш функции можете ориентироваться на дисперсию и стандартное отклонение. 
### Сделайте хэш функцию, дисперсия и стандартное отклонение которой будут стремиться к ~1.
